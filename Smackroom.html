- Smack Room W.I.P -

Concept:

This is Project-SmackRoom, a VR Online Multiplayer Dungeon Crawler that is set in Hollywood, the game it's overall theme is cartoony with the enemies having a chibi look to them. The goal of this game is to progress throughout the level, and to do that you need to kill every single enemy with a weapon of your choice. For this assignment I have a timespan of 8 weeks, and to this day it is still being worked on. This project has also been one of my first real projects that I have been using Photon with. and to combine it with VR has proven to be a real challenge so far, but also the most fun!




What I have done:

I had the role of Lead Developer. The following tasks is where I did the majority/all of the work.

- Networking / Player Instance
- Data Syncing
- Enemy Behaviour and partial implementation
- UI with Animations + Functionality
- Implementation 50%
- Movement
- Weapon Behaviour 90%

Challenges:

This game has proven to be the most difficult project that i've done by far, with the reason being that VR + Networking is a combinations of workflows that i have not yet used up until this project. The data syncing together with the interactions of several objects and detecting which player should perform what event on who's screen has been a big mind boggle. But the longer I have been working on it, the more efficient and understandable the workflow has gotten.


One of the very first difficulties I have encountered in this project thus far has been the Enemies. This is with the reason that the enemies ragdoll and everything that happens on the main screen, should almost perfectly be copied on the rest of the screens without jittering to make it feel clean, but with the limitations of how much data we can synchronize at once, this has proven to be a huge struggle, and has taken a chunk of time to find a viable fix for it. After that I started working on a EnemyManager that would handle every instance of an enemy in a server over the network, so that it's behaviour would be perfectly synchronized across all the clients that had joined the host it's server.
