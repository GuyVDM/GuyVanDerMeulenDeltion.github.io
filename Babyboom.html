Baby-Boom

Tools used:

- Unity

- Visual Studio Code

- Cinemachine Unity Package

- Textmesh Pro



What i worked on this project:

- Player Controller & Mechanim

- Dialogue System

- Sound Implementation

- Cutscenes

- Quests

- AI

- All UI (With exception to the title screen and cosmetics)

This is "Project Baby-Boom", this has been my second project. In this project i had the role of Lead Developer.

This project is a Third-Person Platformer inspired by games like Donkey Kong 64 and Mario 64.

This kind of project is something that was a bit more familiar with the direction i was going with,

and with the timespan of 7 weeks on my and our team's hands we decided to start with the project right of the bat.

The game is about a baby that lives in a world full of candy, with the goal to achieve the Golden Rattle.

This specific game would feature diverse game mechanics that were yet to be explored by me, like: Quests, and a Dialogue system.

And so the project starts.

So the first mechanic that i wanted to tackle were Quests.

The goal was to make this as modulair as it could be, so i could create quests within a short time span.

The way i build the template was with 3 core functions: A function that would start the quest, one that would check the current status, and a reward function.

Since the intent were to make different type of quests, we applied inheritance to it in the case we had extra time to make more types of quests.

In the end we rolled with a type of collectable quest that checked if enough of a specific type of action has been performed, or if the player had enough collectables.

And no quest is a quest without a reward, so we created a function that would reward the player with some candy once the quest has been completed.

Furthermore we got the Dialogue System.

Since we wanted to give the NPC's characteristics, and we wanted the quests objective to be explained, i decided to create a Dialogue System.

This system has been build up really simple, what this system does is basically add a letter to the text UI with an interval of a float that's set beforehand.

And whenever the system saw there were no other characters to be added to the string, it would wait for the player to continue to the next one until there is no more dialogue to be shown.

After that, the script will cut off and the player would be able to move again.

With addition i made the dialogue system in a way so possible functions could be activated after the dialogue if needed.

We used scriptable objects to store diverse functions in a list made out of UnityEvents, and in the scriptable object was one function that would invoke all other stored functions.
