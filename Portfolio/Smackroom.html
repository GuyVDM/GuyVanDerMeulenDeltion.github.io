Smack Room

Tools used:

- Unity

- Photon PUN

- SteamVR Unity Package

- Textmesh Pro



What i worked on this project:

- Networking

- Enemies

- Movement

- UI_Animations

This is "Project Smackroom" and this is my second look into VR development, and this time with online multiplayer.

In this project i had the role of Lead Developer.

This project is a VR Dungeon Crawler, and with this game the goal is to defeat enemies and navigate through the level to find the end.

This project has also been my second time working with Networking, and this seemed as a game that would be alot of fun to play with others.

Since i would be using a networking package, i needed to change my workflow up a bit so the events would be sustained and recreated by all the other clients.

With this type of workflow, i would need a function for every event that needed to happen over the network, and thus needed to be synced across all other clients that are joined.

The first major thing i wanted to program so it would function across the network, were enemies.

This would be one of the most challenging things to program with the reasons that stuff like Animations, and UI needed to be synced.

In the picture shown above, you'll see the EnemyManager, and this manager would handle any action that were to be performed by the enemy,

and would send it over the Network across to all the other linked clients.

Networking also brought with itself, alot of think work about how to locally indentify and handle your character instance, without obstructing other players that may have joined.

Considering our players would needed to be spawned in over the network, and do multiple checks across diverse components to check if it truly is yours to control.

So what i did was store every component a player would need into a local Player Manager, and make it so that once the player had spawned,

that it'd check if the player that spawned in was truly from your client only, after it'd been indentified as your player, it would turn all the components on, and the player would be yours to control.



As of today, this project is still Work in Progress.
